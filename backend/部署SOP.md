# E 組九宮格 後端部署 SOP（給非技術者）

給第一次部署 / 之後改版都照這份走，全程約 10-15 分鐘。

> 鐵律：**這支 GAS 是「任何人（匿名）」都能存取的 web app**，這個存取設定只能在編輯器 UI 手動設，
> 任何形式的 `clasp deploy` 都會把它重置成 403。**開發者不會幫你跑部署這步**，一定是你（或會操作
> 的人）在瀏覽器裡手動點。改版時同理，一律走「編輯現有部署」，絕不要選「新增部署作業」。

---

## 第一次部署（全新建置）

### 1. 開一個新的 GAS 專案

1. 瀏覽器開 [script.new](https://script.new)（會自動登入你目前的 Google 帳號）
2. 出現空白編輯器後，把預設的 `Code.gs` 內容全部刪除
3. 打開這個資料夾的 `Code.gs`（`C:\Users\Joan\Projects\e-team-grid\backend\Code.gs`），全選複製
4. 貼到 GAS 編輯器裡，儲存（Ctrl+S）
5. 專案名稱建議改成「E組九宮格後台」（左上角點一下「無標題專案」可改名）

### 2. 設定 Telegram 通知用的兩個屬性

1. 編輯器左側齒輪圖示「專案設定」
2. 往下找到「指令碼屬性」，點「新增指令碼屬性」
3. 新增兩筆：
   - 屬性：`TG_TOKEN`　值：見 vault `[[敏感憑證總表]]` § Telegram OPS bot（`wuohome_ops_bot` 那一列的 token）
   - 屬性：`TG_CHAT_ID`　值：`8163308207`（Joan 的 Telegram chat id，已在系統其他 bot 沿用）
4. 儲存

> 沒設這兩個屬性也不會壞——`notifyTelegram()` 會偵測到空值直接跳過通知，資料照樣正常寫入，
> 只是妳收不到 Telegram 訊息。之後想開通知，回來補這兩筆屬性即可，不用重新部署。

### 3. 跑 setup() 建表

1. 編輯器上方函式下拉選單，選 `setup`
2. 點「執行」（三角形播放鈕）
3. 第一次跑會跳出 Google 授權視窗——這是正常的（腳本要讀寫你的 Google Sheet）
   - 點「檢閱權限」→ 選你的帳號 → 若出現「Google 尚未驗證這個應用程式」→ 點左下角「進階」→
     點「前往 E組九宮格後台（不安全）」→ 點「允許」
4. 跑完後：
   - 跳出彈窗「✅ 已建立/更新...」
   - 左側「執行紀錄」（或底部「執行項目」）會印出 Sheet 網址，點開確認 15 人資料都在
   - Sheet 名稱是「E組九宮格_後台資料」，會出現在你的 Google Drive 根目錄

> 之後想重新灌一次初始資料（例如 data.js 又更新了、要重新轉一次）：**重新產生 INIT_DATA
> 再貼過去、重跑 setup() 即可**，會清空「E組資料」分頁重建，不影響 Sheet 裡其他你手動加的分頁。
> 但如果夥伴已經在線上編輯過資料，重跑 setup() 會蓋掉他們的修改，**上線後不要再跑 setup()**。

> [!warning] 2026-07-06 修復：id 欄前導零問題
> 舊版 `setup()` 曾發生「003 被 Google Sheet 自動轉成數字 3、前導零消失」，導致 `?id=007`
> 查無資料。新版 `setup()` 會在寫入前**自動把 id 欄整欄鎖成純文字格式**，doGet／doPost 兩端
> 也都會把 id 正規化成三位字串再比對，重跑 setup() 不會再復發。**如果妳的 Sheet 是舊版
> setup() 建的（id 欄看起來是靠右對齊的數字，不是靠左對齊的文字），重跑一次新版 setup()
> 即可修復**，不用手動改儲存格格式。

### 4. 部署成 Web App

1. 編輯器右上角「部署」→「新增部署作業」（**只有這第一次用「新增部署作業」，之後改版一律用「編輯現有」，見下方章節**）
2. 齒輪圖示選「網頁應用程式」
3. 設定：
   - 說明：可填「v1」
   - 執行身分：**我**（你的帳號）
   - 誰可以存取：**任何人**（注意不是「擁有 Google 帳戶的任何人」，要選最寬鬆那個「任何人」）
4. 點「部署」，可能再跳一次授權視窗，同第 3 步驟操作
5. 完成後會出現一個網址，長得像：
   `https://script.google.com/macros/s/AKfycb.../exec`
6. **把這個網址完整複製起來**，交給 developer 填進前端 `index.html` 的 `API_URL` 常數

### 5. 驗證部署成功

用瀏覽器直接開剛剛那個 `/exec` 網址，應該看到一串 JSON，開頭像：
```json
{"ok":true,"data":{"003":{...
```
看到 `"ok":true` 就是成功了。如果看到「登入」畫面或 403，代表第 4 步「誰可以存取」沒選對，回去
「管理部署作業」把它改回「任何人」。

**務必額外檢查 id 前導零**：JSON 裡的 key 一定要是三位數字字串（`"003"`、`"007"`），不能是
`"3"`、`"7"` 這種掉了前導零的樣子。再開一次 `/exec?id=007` 單獨測，確認回的是張瓊安（而不是
`{"ok":false,"error":"找不到 id：007"}`）。這兩項沒過，代表舊版 code 還沒換掉，回頭確認第 1
步是不是真的貼了最新版 `Code.gs`。

## 之後改版（改 Code.gs 內容後要重新上線）

**絕對不要點「新增部署作業」**——那會產生新網址，前端 `index.html` 裡填的舊網址會卡在舊版程式碼，
造成「手動測試看得到新功能、夥伴用起來卻是舊行為」的詭異狀況（GAS 已知地雷，踩坑速查有記錄）。

正確步驟：

1. 編輯器貼上新版 `Code.gs` 內容，儲存
2. 右上角「部署」→「管理部署作業」
3. 找到目前線上那個部署（通常只有一個），點鉛筆圖示「編輯」
4. 版本選「新版本」
5. 「誰可以存取」**再檢查一次是不是「任何人」**（GAS 偶爾會在這步被重置，務必肉眼確認一次再送出）
6. 點「部署」

網址不會變，前端不用改，重新整理頁面就會吃到新程式碼。

## 疑難排解

| 症狀 | 原因 | 處理 |
|------|------|------|
| 前端顯示「載入失敗」、`/exec` 網址開起來要求登入或 403 | 「誰可以存取」被重置成「僅自己」 | 管理部署作業 → 編輯現有 → 誰可以存取改回「任何人」→ 部署 |
| 存檔後沒收到 Telegram 通知，但資料確實有存進 Sheet | `TG_TOKEN`/`TG_CHAT_ID` 沒設或設錯 | 回專案設定 → 指令碼屬性檢查兩個值 |
| 執行 setup() 跳出一堆紅字 authorization error | 授權視窗被跳過或取消 | 重新點執行，這次耐心跑完「檢閱權限」流程 |
| 想確認目前線上是不是最新版 | — | 開 `/exec?id=007` 網址，看 `updated_at`／內容是否符合預期 |

## 給 developer 的技術備忘（非必要 Joan 看）

- Sheet 分頁名稱固定 `E組資料`，欄位順序 `id / name / updated_at / last_editor / data_json`
- `data_json` 是整份 `{self, themes}` JSON 字串，前端 `doGet` 拿到後直接展開成 `{id, name, updated_at, last_editor, self, themes}`
- `doPost` 用 `text/plain` content-type 收（前端故意不設 `application/json` 避免 CORS preflight，GAS 經典限制）
- Telegram 失敗 catch 在 `doPost` 內部，不會讓整個請求回 500
- `curl` 驗不了 `doPost`（302 一次性 echo URL），要驗證寫入請走真實前端（本機開 `index.html` 填 mock API_URL）或直接在 Sheet 裡人工核對
- id 欄前導零型別坑（003→3）已修復，處理方式與根因見 [[技術筆記/踩坑速查]] §「Google Sheet 把『2026-08』偷轉成日期數字」同型問題；`Code.gs` 的 `normalizeId()` 是本系統的雙保險實作（寫入端 `setNumberFormat('@')` 鎖格式＋讀寫兩端一律正規化再比對）
